# DECISIONS.md

## Why I Chose This Stack

I chose HTML, CSS, and JavaScript for the frontend because the project requirements were simple and these technologies allow quick development without additional frameworks.

For the backend, I used Node.js with Express.js because Express provides a lightweight and structured way to build REST APIs. It also integrates well with MySQL and is easy to maintain.

I selected MySQL as the database because the application contains structured relational data such as users, events, and registrations.

---

## One Decision Not Specified in the Brief

I implemented JWT-based authentication for login sessions. This allows API endpoints to be protected and provides a cleaner authentication approach compared to using simple localStorage flags.

---

## One Thing I Would Improve With More Time

Given more time, I would improve the user interface and add role-based route protection on the frontend. I would also deploy the application so that it could be accessed online without requiring local setup.
